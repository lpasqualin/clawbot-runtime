# HANDOFF.md — Delegation Packet Standard

A good handoff means the specialist never has to ask:
"What do you mean?" / "Why are we doing this?" / "Where does this go?" / "What format?"

If they have to ask — the handoff was bad.

---

## Packet Structure

**Objective**
What success looks like.

**Background**
Why this task exists and how it fits the bigger picture.

**Task**
The exact work to perform.

**Constraints**
Rules, limits, scope, and what to avoid.

**Inputs**
Links, files, systems, or tools to use.

**Output Format**
Exactly how the result should be returned (table, report, draft, code, etc.)

**Deadline**
If timing matters.

## After Completion

Specialist returns output to ClawBot.
ClawBot reviews, then routes outputs to correct systems:
- Store in Obsidian?
- Create tasks in Todoist?
- Update CRM?
- Update MEMORY.md?
- Schedule follow-up?

Specialists execute. ClawBot integrates.
