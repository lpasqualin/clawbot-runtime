# DELEGATION_TEMPLATE.md — Standard Delegation Packet

This template is used by ClawBot when delegating work to a specialist agent.

Core rule:
ClawBot decides, specialists execute, systems of record store truth, Leo approves major decisions and irreversible external actions when required.

---

## Delegation Packet

### Project
Which project this belongs to.

### Task
What needs to be done.

### Why
Why this task matters in the larger context or business objective.

### Status
New / In Progress / Blocked / Review

### Context
Relevant background only.
Include prior work, links, constraints, assumptions, and known information.
Do NOT include entire project history — only what is needed.

### Constraints
Boundaries, exclusions, preferences, and what to ignore.
What the agent is NOT allowed to do.
What the agent should avoid.

### Output
Exact deliverable required.
Be specific about format:
- List
- Report
- Draft
- Table
- File
- Code
- Notes written to a specific system

### Open Questions
Anything unclear that may affect execution.
If none, write: None.

### Recommended Next Step
What should happen after this task is completed.

### Approval Needed
Yes / No  
If yes, explain why approval is required.

---

## Definition of Done

The task is complete when:
- The Output is produced in the correct format
- It is stored in the correct system of record
- Open Questions are listed
- Recommended Next Step is proposed
- Approval status is clearly stated

If these are missing, the task is not done.
